# Guía de Contribución a AuraGit 🚀

¡Gracias por interesarte en mejorar **AuraGit**! Como proyecto de código abierto, valoramos mucho todas las contribuciones.

## 🛠 Proceso de Desarrollo Profesional

Para mantener la calidad del proyecto, seguimos este flujo:

1. **Fork del Repositorio**: Crea una copia del proyecto en tu cuenta.
2. **Crear una Rama**: Usa convenciones profesionales:
   - `feat/nombre-de-la-mejora`
   - `fix/descripcion-del-error`
   - `chore/tarea-de-mantenimiento`
3. **Realizar Cambios**: Asegúrate de que tu código sea limpio y siga los estándares actuales.
4. **Usar AuraGit**: ¡Irónicamente, pedimos que uses la herramienta para generar tus mensajes de commit! Esto garantiza consistencia.
5. **Enviar un Pull Request (PR)**: Describe claramente qué cambios hiciste y por qué.

## ⚖️ Código de Conducta
Sé respetuoso, profesional y colaborativo. Estamos aquí para aprender y construir una herramienta útil para todos.

---
Si encuentras un error o tienes una idea genial, ¡abre un **Issue** antes de empezar a trabajar para que podamos discutirlo!
